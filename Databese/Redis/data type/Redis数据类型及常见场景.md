4.	Hash数据类型
4.1.	数据结构
就像关系型数据的一张表，key是唯一的，而value是一个hashmap的数据结构，所以可以像关系型数据库一样操作value中的某个字段。
4.2.	应用场景
存储对象数据结构，可以任意添加或删除field，并且存储对象时不需要对数据进行转换和解析
5.	List数据类型
5.1.	数据结构
一个双向链表，可以在头部或尾部进行插入或弹出。
5.2.	应用场景
5.2.1.	消息队列
5.2.2.	最新内容
5.2.3.	列表旋转
6.	Set数据类型
6.1.	数据结构
也是一个hash表，只是value值总为null。可以用来插入或删除键。并判断一个键是否存在，求交集、并集、差集。
6.2.	应用场景
6.2.1.	过滤器，白名单、黑名单
6.2.2.	共同好友，求交集可实现。
